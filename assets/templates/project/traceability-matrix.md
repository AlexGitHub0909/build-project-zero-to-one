# Traceability matrix

Use stable requirement IDs. Keep target behavior and current evidence in separate columns.

| Requirement ID | Target behavior | Flow | Interface/API | Data/state | Implementation | Test evidence | Status | Gap or next action |
|---|---|---|---|---|---|---|---|---|

## Rules

- `IMPLEMENTED` requires current implementation and evidence.
- `PARTIAL` names the missing behavior or evidence.
- `TODO / GAP` is not a release pass.
- `MANUAL_REQUIRED` names the owner and procedure.
- `BLOCKED` names the dependency and unblock condition.
- `FORBIDDEN / OUT_OF_SCOPE` remains visible so it is not implemented by accident.
